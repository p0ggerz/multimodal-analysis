# Мультиязыковое исследование лингвистических особенностей заголовков постов в социальных сетях (на примере русскоязычного сегмента Pinterest и китайской платформы Xiaohongshu)

## Описание

**Цель проекта – выявление и сравнительный анализ лингвистических особенностей заголовков постов для визуально-ориентированных платформ Pinterest и Xiaohongshu. Анализ охватывает графический, фонетический, морфологический, лексико-семантический, синтаксический уровни и лингвопрагматику.**

## Структура проекта

| Файл | Содержание |
|---|---|
| `протокол_разметки.xlsx` | **«Протокол разметки»** – 100 русскоязычных и 100 китаеязычных заголовков по 75 параметрам, сгруппированным в 6 блоков; **«Морфологическая разметка»** – результаты автоматической частеречной разметки каждого заголовка: текст заголовка, язык, полный частеречный состав, а также преобладающая часть речи в английской и русской нотации |
| `Pinterest_Data.xlsx` | **«Pinterest Data»** – все 635 постов, полученных в ходе парсинга, с метаданными (`id`, поисковое слово, автор, число подписчиков, тип публикации, заголовок, описание, дата, URL, домен); **«Keywords»** – ключевые слова, по которым производился сбор данных, и количество результатов по каждому; **«Итог»** – отобранные 100 заголовков с основными полями (`id`, поисковое слово, заголовок, дата) |
| `linguistic_analysis.ipynb` | Код статистического анализа: частотные таблицы, IPM, корреляция Спирмена, U-критерий Манна–Уитни с поправкой Беньямини–Хохберга, визуализация результатов (среда – Google Colab, язык – Python 3.11) |

## Методология

**Корпус:** 200 заголовков (по 100 на каждый язык), год публикации – 2021.  
**Токенизация:** для русского – пробельная разбивка (слово = токен), для китайского – подсчёт иероглифов CJK.  
**Метрика:** IPM (instances per million tokens) для нормировки корпусов разного объёма.  
**Статистика:** корреляция Спирмена по IPM-векторам блока, U-критерий Манна–Уитни по нормированным частотам, поправка Беньямини–Хохберга (FDR = 5%) на 75 одновременных тестов.

## Требования

Для запуска необходимо загрузить файл `протокол_разметки.xlsx` (первая ячейка запускает диалог загрузки в Colab).
